Project Overview

An introduction to the OUP Test Generator web application, its purpose, and scope as defined in the SRS document.

Purpose

To define the requirements for the development of a Test Generator web application for Oxford University Press (OUP) Pakistan. This application will serve a B2B model for schools and is designed for future expansion into a B2C model and integration with an LMS.

Scope

The initial scope is limited to the B2B model, defining functional requirements for roles like Admin, Teacher, and Student. Core functionalities include user management, question bank creation, test generation (online/offline), and performance analytics.

Core Product Functions

  • ✔️
    Manage Users: Super Admin can create, delete, and manage user roles and rights.
  • ✔️
    Create Question Banks: Content Creators and Teachers can create questions, with teacher submissions requiring approval.
  • ✔️
    Generate Tests: Teachers can create tests based on various criteria in both online and offline modes.
  • ✔️
    View Statistics: Various roles can view detailed statistics on user activity and performance.
  • ✔️
    Student Features: Students can attempt assigned quizzes and view their performance.

Development Approach & Timeline

This section outlines the modern agile development process for the OUP Test Generator, with a high-level timeline to ensure project completion by the target date of January 2026. This approach prioritizes UI/UX design before coding to ensure a user-centric final product.

1. Requirement Gathering (Aug-Sep 2025)

The process begins with a deep dive into the business goals, user needs, and technical constraints. This phase involves stakeholders, product managers, and end-users to create a definitive requirements document and user stories.

2. UI/UX Design Phase (Sep-Oct 2025)

This is where the look and feel of the application are crafted.
UX Design: focuses on user flows, information architecture, and wireframes.
UI Design: focuses on the visual aspects, including colors, typography, and interactive elements. This phase is critical because it guides all subsequent coding efforts and ensures alignment with user expectations.

3. Frontend & Backend Development (Oct-Dec 2025)

This is the core coding phase.
Developer will translate the UI designs into interactive interfaces and build the server-side logic, databases, and APIs.

4. Testing (Dec 2025)

The application will undergo rigorous testing, including Unit Testing, Integration Testing, UI Testing, and User Acceptance Testing (UAT). This ensures the app works as intended, is bug-free, and meets all specified requirements before launch.

5. Deployment & Launch (Jan 2026)

The final tested version of the application will be deployed to the production environment. Continuous Integration and Continuous Deployment (CI/CD) pipelines will be used to automate this process, leading to a smooth launch.

6. Maintenance & Iteration (Post-Jan 2026)

Based on user feedback, analytics, and performance metrics, the team will continue to make updates and improvements, ensuring the application remains a valuable tool.
